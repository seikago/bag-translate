# Bag Translate — MVP 产品需求文档

**版本：** v1.0
**日期：** 2026-03-26
**作者：** 星卡

---

## 一、产品概述

### 背景
包袋行业外贸从业者在与日本客户沟通时，缺乏专业的日语口语练习工具。现有日语学习产品（如多邻国、HelloTalk）均为通用场景，无法满足包袋行业的专业词汇和商务沟通需求。

### 产品定位
**Bag Translate** 是一款专为包袋行业外贸人设计的日语口语练习工具，通过 AI 模拟真实客户对话场景，帮助用户快速掌握行业专业日语表达。

### 目标用户
- 包袋行业外贸业务员（主要）
- 有意开拓日本市场的外贸从业者
- 需要提升日语商务沟通能力的人群

### 核心价值
> 不是学日语，是学"包袋外贸日语"

---

## 二、MVP 范围

MVP 阶段只做最核心的功能，验证用户需求，快速上线。

**MVP 目标：** 3 个核心场景 + AI 对话 + 词汇库，2 周内可上线

---

## 三、功能需求

### 3.1 核心功能（Must Have）

#### F1 — 场景化对话练习

用户选择一个练习场景，与 AI 扮演的"日本客户"进行对话练习。

**MVP 阶段包含 3 个场景：**

| 场景编号 | 场景名称 | 说明 |
|---|---|---|
| S01 | 产品介绍 | 向客户介绍包袋产品规格、材质、工艺 |
| S02 | 询价报价 | 处理客户询价、报价、MOQ 谈判 |
| S03 | 质量问题处理 | 应对客户投诉、质量异议、售后处理 |

**交互流程：**
1. 用户选择场景
2. 系统展示场景背景说明（中文）
3. AI 客户发起第一句对话（日语）
4. 用户输入回复（文字输入，MVP 阶段不做语音）
5. AI 给出回应 + 对用户回复的点评（语法、用词建议）
6. 对话持续 5-8 轮后结束，给出总体评分和建议

**AI 角色设定（Prompt 核心）：**
- 角色：日本包袋采购负责人（如 ELECOM 采购部）
- 性格：专业、礼貌、有时会提出价格压力或质量疑虑
- 语言：标准商务日语，适当使用行业术语
- 评分维度：礼貌程度、专业词汇使用、表达准确性

---

#### F2 — 专业词汇库

包袋行业日语专业词汇查询和学习。

**MVP 阶段词汇量：** 100-150 个核心词汇

**词汇分类：**

| 分类 | 示例 |
|---|---|
| 材质面料 | ナイロン（尼龙）、ポリエステル（涤纶）、PU レザー（PU皮） |
| 五金配件 | ファスナー（拉链）、バックル（扣具）、Dカン（D型环） |
| 产品类型 | バックパック（双肩包）、トートバッグ（托特包）、スーツケース（行李箱） |
| 工艺术语 | 縫製（缝制）、裏地（内衬）、刺繍（刺绣） |
| 商务用语 | 納期（交期）、最小ロット（MOQ）、サンプル（样品） |

**功能：**
- 搜索词汇（中文/日文均可搜索）
- 查看词汇详情（读音、例句、使用场景）
- 收藏词汇，生成个人词汇本

---

#### F3 — 每日一句

每天推送一条包袋行业常用日语表达，培养用户使用习惯。

**内容格式：**
- 日语原文
- 中文翻译
- 使用场景说明
- 真实案例（如"这句话在展会上经常用到"）

---

### 3.2 次要功能（Should Have，MVP 后期迭代）

| 功能 | 说明 |
|---|---|
| 语音输入 | 用户用日语说话，系统识别并评分 |
| 练习记录 | 记录用户历史对话，可回顾复习 |
| 打卡系统 | 连续练习天数统计，激励用户坚持 |
| 更多场景 | 展会接待、邮件写作、电话沟通等 |

---

### 3.3 不做（Not in MVP）

- 用户注册/登录系统（MVP 阶段无需账号）
- 社区功能
- 付费订阅
- 移动端 App
- 多语言支持（只做日语）

## 四、技术方案

### 技术栈

| 层级 | 技术选型 | 理由 |
|---|---|---|
| 前端 | Next.js 14 + Tailwind CSS | 快速开发，SEO 友好 |
| AI 对话 | Claude API（claude-sonnet） | 日语能力强，理解行业上下文 |
| 部署 | Vercel | 免费，一键部署 |
| 数据存储 | 本地 JSON 文件（词汇库） | MVP 阶段无需数据库 |

### 系统架构（MVP 极简版）

```
用户浏览器
    ↓
Next.js 前端（Vercel）
    ↓
/api/chat（Next.js API Route）
    ↓
Claude API（AI 对话 + 点评）
```

### AI Prompt 设计（核心）

```
你是一位日本包袋采购负责人，正在与中国供应商进行商务沟通。

角色设定：
- 公司：日本知名电子配件品牌采购部
- 职位：采购负责人
- 性格：专业、礼貌，偶尔会提出价格压力或质量疑虑
- 语言：使用标准商务日语

当前场景：[场景描述]

对话规则：
1. 用日语与用户对话
2. 每次对话后，用中文给出对用户回复的简短点评
3. 点评格式：【点评】语法/用词建议...
4. 对话进行 5-8 轮后，给出总体评分（满分10分）和改进建议
```

## 五、页面设计

### 页面结构（共 4 个页面）

```
首页（/）
├── 产品介绍
├── 3 个场景入口
└── 词汇库入口

对话练习页（/practice/[scene]）
├── 场景说明
├── 对话区域
├── 输入框
└── AI 点评区

词汇库页（/vocabulary）
├── 搜索框
├── 分类筛选
└── 词汇列表

词汇详情页（/vocabulary/[id]）
├── 词汇信息
├── 例句
└── 收藏按钮
```

### UI 风格
- 简洁、专业，偏商务感
- 主色调：深蓝 + 白色（参考日系商务风格）
- 字体：中文用思源黑体，日文用系统默认日语字体

## 六、上线计划

| 阶段 | 时间 | 目标 |
|---|---|---|
| 第 1 周 | Day 1-3 | 搭建项目框架，完成首页和对话页 |
| 第 1 周 | Day 4-5 | 接入 Claude API，完成 AI 对话功能 |
| 第 2 周 | Day 6-8 | 完成词汇库页面，录入 100 个词汇 |
| 第 2 周 | Day 9-10 | 测试、优化、部署上线 |
| 上线后 | Week 3-4 | 收集用户反馈，决定下一步迭代方向 |

## 七、成功指标（MVP 验证标准）

| 指标 | 目标值 | 说明 |
|---|---|---|
| 上线后 2 周访问量 | ≥ 500 UV | 通过公众号推广 |
| 用户完成一次完整对话 | ≥ 30% | 验证核心功能吸引力 |
| 用户回访率（7日） | ≥ 20% | 验证产品粘性 |
| 用户反馈收集 | ≥ 20 条 | 用于指导下一步迭代 |

---

## 八、风险与应对

| 风险 | 概率 | 应对方案 |
|---|---|---|
| Claude API 费用超预期 | 中 | 设置每日调用上限，MVP 阶段限制对话轮数 |
| 用户不愿意打字练习（想要语音） | 高 | 快速迭代加入语音功能，或先用语音输入作为 MVP |
| 词汇库内容不够专业 | 低 | 基于你的实际工作经验整理，质量有保障 |
| 推广效果不佳 | 中 | 先在 AI 破局俱乐部内测，收集真实反馈 |
